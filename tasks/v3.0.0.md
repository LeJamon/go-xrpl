# v3.0.0 — rippled 3.0.0 parity (integration branch)

Tracking branch for goXRPL's update to **rippled 3.0.0** parity (tag `3.0.0`,
branched from `2.6.2` — 161 commits). Parent issue: #274.

`main` stays at **2.6.2** parity (used for live testnet interop with rippled
2.6.2). All v3.0.0 work lands on this `v3.0.0` branch via per-phase PRs, and
this branch merges into `main` once 3.0.0 parity is complete and verified.

## Working model

- Feature PRs for each phase target **`v3.0.0`**, not `main`.
- This branch is the integration point; keep it green (`just test`,
  `just conformance`) as phases land.
- When all phases are complete and verified (Phase G), open/flip the umbrella
  PR `v3.0.0 → main` out of draft.

## Confirmed NOT changed in 3.0.0

Peer wire protocol (`ripple.proto`, formatting only), consensus
messages/algorithm, TER codes (up to `tecNO_DELEGATE_PERMISSION = 198`), ledger
header format, validator manifest format, STAmount wire encoding.

## Deferred to v3.1.0 (separate tracking issue)

(40+ new sfields); 11 `Loan*` transactors; `keylet::loanbroker` / `keylet::loan`
(namespace bytes `'l'` / `'L'`); `Units.go` bips helpers; `DynamicMPT`
transactor changes; `fixDelegateV1_1` permission-delegation restrictions.
